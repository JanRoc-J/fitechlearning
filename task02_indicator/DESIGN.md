**版本**: v0.1-draft | **日期**: 2026-07-09 | **状态**: 设计阶段

## 1. 产品概述

### 1.1 一句话定位

一个纯前端 HTML 工具，用户选择股票 → 调整四个指标的参数 → 实时重绘图表，无需后端服务。

### 1.2 核心价值

| 痛点 | 解决方案 |
|:---|:---|
| Notebook 分析需要改代码改参数 | 滑块 (slider) 拖拽即重算重绘 |
| 不同股票难以快速对比 | 下拉切换，数据秒级加载 |
| 参数调优没有视觉反馈 | 每次参数变化，50ms 防抖后自动重绘 |
| 分析结果难以分享 | URL hash 保存状态，复制链接即可分享 |

### 1.3 技术选型

| 层 | 选型 | 理由 |
|:---|:---|:---|
| 框架 | 零框架 (Vanilla JS) | 无依赖，一个 HTML 文件即可运行 |
| 图表 | ECharts 5.5 (CDN) | 交互丰富、性能好、中文生态成熟 |
| 数据 | 预置 JSON (内嵌) | 离线可用，避免 CORS 问题 |
| 布局 | CSS Grid | 自适应，无需引入 CSS 框架 |
| 数据源 | Tushare Pro (预取) | 日线数据 + 指标计算全在浏览器完成 |

## 2. 架构设计

### 2.1 整体数据流

```
┌──────────────┐     ┌─────────────────┐     ┌──────────────┐
│  数据层       │ ──→ │   计算引擎       │ ──→ │   视图层      │
│  (JSON 内嵌)  │     │  (纯 JS 函数)    │     │  (ECharts)   │
└──────────────┘     └─────────────────┘     └──────────────┘
       ↑                      ↑                     │
       │                      │                     │
  ┌────┴─────┐          ┌─────┴──────┐        ┌─────┴──────┐
  │ Stock     │          │ Parameter   │        │ Render     │
  │ Selector  │          │ Sliders     │        │ Queue      │
  └───────────┘          └─────────────┘        └────────────┘
                              │
                         用户交互
```

### 2.2 模块划分

| 模块 | 文件/位置 | 职责 |
|:---|:---|:---|
| `DataStore` | `data/` JS 对象 | 预置多个股票的 OHLCV 数据，按 `ts_code` 索引 |
| `IndicatorEngine` | `indicators.js` | 纯函数：接收 `prices[]` + `params{}` → 返回指标序列 |
| `ChartManager` | `charts.js` | 管理 4 个 ECharts 实例，接收数据 → setOption 渲染 |
| `UIController` | `app.js` | 绑定滑块/选择器事件 → 触发防抖重算 → 通知 ChartManager |
| `StateManager` | `state.js` | 维护当前 `{stock, params, dateRange}`，同步到 URL hash |

### 2.3 页面加载流程

```
1. HTML 加载 → 内嵌 JSON 数据解析 → DataStore 初始化
2. 读取 URL hash → 恢复上次参数 (若无则用默认值)
3. UI 渲染控制面板 + 4 个图表容器
4. 首次计算 → 首次渲染
5. 绑定事件: slider input → 50ms debounce → 重算 → 重绘
```

## 4. 参数系统

### 4.1 所有可调参数

| 指标 | 参数 | 默认值 | 范围 | 步长 | 说明 |
|:---|:---|:---:|:---:|:---:|:---|
| **RSI** | period | 14 | 2-50 | 1 | 平滑周期 |
| | overbought | 70 | 50-95 | 1 | 超买阈值线 |
| | oversold | 30 | 5-50 | 1 | 超卖阈值线 |
| **MACD** | fast | 12 | 2-30 | 1 | 快线 EMA 周期 |
| | slow | 26 | 5-60 | 1 | 慢线 EMA 周期 |
| | signal | 9 | 2-20 | 1 | DEA 信号线周期 |
| **Bollinger** | period | 20 | 5-50 | 1 | 中轨 SMA 周期 |
| | multiplier | 2.0 | 1.0-4.0 | 0.1 | 标准差倍数 |
| **ATR** | period | 14 | 2-50 | 1 | 平滑周期 |

### 4.2 参数变更行为

```
slider.oninput → 50ms debounce → recalcAll() → setOption (notMerge:true) → URL hash update
```

- **防抖**: 50ms，避免滑块拖拽时每帧都重算
- **非合并更新**: `setOption(opt, {notMerge: true})` 确保旧数据完全替换
- **URL 持久化**: 参数变化后 `window.location.hash = encode(params + stock)`，方便分享

### 4.3 重置 vs 应用

| 按钮 | 行为 |
|:---|:---|
| **重置默认** | 所有滑块回到默认值，触发重算重绘 |
| **应用计算** | 等同于滑块变动后的自动行为（显式触发按钮给触摸屏用户） |

## 7. 计算引擎设计

### 7.1 函数签名 (TypeScript 伪代码)

```typescript
interface IndicatorParams {
  rsi:    { period: number; overbought: number; oversold: number };
  macd:   { fast: number; slow: number; signal: number };
  boll:   { period: number; multiplier: number };
  atr:    { period: number };
}

interface IndicatorResults {
  rsi:    number[];           // RSI values array
  macd:   { dif: number[]; dea: number[]; bar: number[] };
  boll:   { mid: number[]; upper: number[]; lower: number[] };
  atr:    number[];           // ATR values array
}

function computeAll(prices: OHLCV[], params: IndicatorParams): IndicatorResults;
```

### 7.2 性能考量

| 操作 | 复杂度 | 10年数据 (2500条) 耗时 |
|:---|:---|:---:|
| RSI 计算 | O(n) | < 1ms |
| MACD 计算 | O(n) | < 1ms |
| Bollinger 计算 | O(n×period) | ~5ms |
| ATR 计算 | O(n) | < 1ms |
| ECharts setOption | — | ~20ms |
| **总计 (单次)** | — | **< 50ms** |

浏览器主线程完全可承受实时重算。

## 10. 边界与约束

- **仅支持日线数据** — 分钟线数据量大，建议后续用 Web Worker 计算
- **数据需预取** — 当前不内置 Tushare MCP 实时取数（受 CORS 限制），数据刷新需要手动更新 HTML
- **浏览器兼容** — 目标 Chrome / Edge / Firefox 最新版，不保证 IE 兼容
- **不提供回测** — 本工具只做指标计算和可视化，不做策略回测
